# 회차 입찰 라이프사이클 (서칭뷰 & 신제품검색)

> 작성일: 2026-05-27
>
> NOSP의 회차별 입찰 흐름과 대시보드에서의 상태 정규화 룰을 정의합니다.

## 1. 회차 1개의 흐름

각 회차는 **약 4주에 걸친 7단계**로 진행됩니다 (서칭뷰·신제품검색 동일, 기념일 광고는 일정이 일부 다름).

```
정기입찰 ─→ 정기 발표 ─→ 재입찰 ─→ 재입찰 발표 ─→ 집행 대기 ─→ 집행 ─→ 종료
  (7일)     (1일)       (5일)      (1일)        (5일)        (7일)
```

### 1.1 단계별 상세

| # | 단계 | 기간 (요일) | 입찰 상태 | 비고 |
|---|------|-------------|-----------|------|
| ① | **정기입찰기간** | 화~월 (약 7일) | `입찰가능` | 모든 광고주 자유 입찰 |
| ② | **정기입찰 발표일** | 화 (1일) | (전환일) | 09시 결과 발표 |
| ③ | **재입찰기간** | 수~월 (약 5일) | `입찰가능` (공실 있을 때) / `입찰중지` (공실 0) | 발표 직후 공실 발생 시에만 진행 |
| ④ | **재입찰 발표일** | 화 (1일) | (전환일) | 09시 결과 발표 |
| ⑤ | **집행 대기** | 수~일 (약 5일) | **`입찰기간종료`** | 모든 입찰 종료, 집행 시작 전 |
| ⑥ | **집행기간** | 월~일 (7일) | **`입찰기간종료`** | 광고 실제 노출 진행 중 |
| ⑦ | **집행 종료 후** | 일 이후 | **`입찰기간종료`** | 완전 종료 |

### 1.2 실제 예시 (회차 202624)

| 항목 | 일자 |
|------|------|
| 정기입찰 기간 | 2026.05.19 ~ 2026.05.25 |
| 정기입찰 발표일 | 2026.05.26 |
| 재입찰 기간 | 2026.05.27 ~ 2026.06.01 |
| 재입찰 발표일 | 2026.06.02 |
| 집행 기간 | 2026.06.08 ~ 2026.06.14 |

오늘 날짜에 따라 표시되는 상태:

| 오늘 날짜 | 상태 |
|-----------|------|
| ~ 5/25 | `입찰가능` (정기) |
| 5/26 | 발표일 |
| 5/27 ~ 6/01 | `입찰가능` (재입찰, 공실 있을 때) 또는 `입찰중지` |
| 6/02 | 재입찰 발표일 |
| **6/03 이후** | **`입찰기간종료`** (집행 대기 + 집행 + 종료 모두 포함) |

## 2. 핵심 정규화 룰 ⭐

DB의 `bid_status` 컬럼은 **CSV 다운로드 시점의 NOSP 상태**이므로 시간이 지나면 stale 됩니다 (예: 4월에 다운로드한 CSV에 "입찰가능"으로 기록된 회차가 5월에는 이미 종료된 상태).

### 룰

```
오늘 > rebid_announce_date          → bid_status = '입찰기간종료'
오늘 > regular_announce_date (재입찰 발표일 없을 때) → bid_status = '입찰기간종료'
```

즉, **재입찰 발표일까지 지난 회차는 무조건 `입찰기간종료`** 로 표시.

### SQL

```sql
UPDATE round_keyword_groups rkg
SET bid_status = '입찰기간종료', updated_at = now()
FROM rounds r
WHERE rkg.round_id = r.id
  AND COALESCE(r.rebid_announce_date, r.regular_announce_date) < CURRENT_DATE
  AND rkg.bid_status IS DISTINCT FROM '입찰기간종료';
```

## 3. 적용 시점

이 정규화는 다음 시점마다 자동 실행됩니다:

1. **매주 CSV ingest 직후** — `worker/src/worker/ingest.py` 의 `_ingest_bid_info()` 끝부분에서 호출
2. **대시보드 표시 직전** — 추가 안전망 (선택, 필요 시 view로 처리)

## 4. 다른 상태 값 참고

| NOSP bid_status | 의미 | 정규화 후 |
|-----------------|------|-----------|
| `입찰가능` | 정기 또는 재입찰 진행 중 | 회차가 끝났으면 `입찰기간종료` |
| `입찰중지` | 재입찰 기간이지만 공실 0개 → 실질적으로 정지 | 회차가 끝났으면 `입찰기간종료` |
| `입찰기간종료` | 모든 입찰 종료 | 그대로 |
