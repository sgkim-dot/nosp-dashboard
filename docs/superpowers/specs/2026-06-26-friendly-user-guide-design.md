## 작업 단계

1. `apps/dashboard/public/guide/` 폴더 생성
2. 스크린샷 11장 자동 캡쳐 (사용자가 localhost:3100 에 로그인된 Chrome 창 유지)
3. 다이어그램 3장 SVG 작성 → PNG export
4. `docs/user-guide.md` 갱신
   - 퀵스타트 챕터 추가 (친근체)
   - 페이지별 매뉴얼에 스크린샷·다이어그램 markdown 이미지 삽입
   - 용어 사전 약간 풀어서 설명
5. 사용자 1차 검수 (이미지 교체 요청 수용)
6. `pnpm docs:sync` → Notion 갱신
7. commit + push

## 호스팅 동작 원리

- 이미지는 `apps/dashboard/public/guide/*.png` 저장
- Vercel 배포 시 `https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/xxx.png` 로 자동 노출
- markdown 안에서는 절대 URL로 참조 → Notion에 sync될 때 외부 이미지로 그대로 표시
- gitignore 없음 (이미지 자체는 공개 가능 — 회사 정보 노출 없도록 캡쳐 시 검토)

## 보안 / 캡쳐 시 주의

- 화면에 실제 입찰 단가, 사업자 이름이 그대로 노출됨
- 캡쳐 후 사용자가 검수 단계에서 **블러 처리·교체**를 요청할 수 있음 (자산 교체 가능하도록 파일명 고정)
- 캡쳐는 일반적인 KG 1개를 표본으로 사용 (어느 광고주 KG를 표본으로 쓸지는 사용자에게 1차 검수 시 확인)
