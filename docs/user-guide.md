# 1. 5분 만에 시작하기

> 일단 이 5단계만 따라해 보세요. 대시보드의 80%를 쓸 수 있게 됩니다.

## ① 로그인하기

대시보드 URL에 접속하면 로그인 화면이 떠요. **회사 이메일**로 가입/로그인하면 됩니다.

![로그인 화면](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/login.png)

> 💡 처음 가입할 때는 `@madup.com` 같은 회사 이메일을 쓰세요. 외부 이메일은 승인이 별도로 필요할 수 있어요.

## ② 사이드바에서 메뉴 선택

로그인하면 왼쪽에 사이드바가 뜹니다. 메뉴는 **3개** 뿐이에요. 어렵지 않죠?

![사이드바](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/sidebar.png)

가장 자주 쓰는 건 맨 위 **입낙찰 히스토리** 입니다. 일단 이거 클릭!

## ③ "빠른 검색" 으로 키워드그룹 찾기

화면 위쪽 필터바 우측에 **빠른 검색** 입력칸이 있어요. 여기에 키워드그룹 이름을 입력해 보세요.

![빠른 검색](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/quick-search.png)

예를 들어 `보험` 이라고 치면 자동완성으로 관련 키워드그룹이 쭉 떠요. ↑ ↓ 키로 이동하거나, 마우스로 클릭해서 골라요.

> 💡 키워드그룹(KG) = 비슷한 검색 의도를 가진 키워드들의 묶음이에요. 예: "자동차보험", "운전자보험" 등.

## ④ 추천 입찰가 보기

키워드그룹을 고르면, 잠깐의 로딩 후 데이터가 가득 채워진 화면이 뜹니다.

![입낙찰 히스토리 전체](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/bid-overview.png)

**가장 중요한 카드**는 우측 상단의 **추천 입찰가** 입니다. 다음 회차에 얼마를 써야 할지 알려줘요.

![추천 입찰가 카드](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/recommend-card.png)

이렇게 나옵니다:

- **추천 입찰가 (VAT-)**: 예 — `5,200원 ~ 5,800원` (이 범위 안에서 결정)
- **과거 적중률**: 같은 전략을 과거에 적용했을 때 얼마나 잘 맞췄는지
  - 🟢 80% 이상 → 그냥 따라가도 돼요
  - 🟡 60~80% → 참고하세요
  - 🔴 60% 미만 → 신중히 검토 필요

> 💡 과거 적중률이 높으면, 자신 있게 추천가 그대로 쓰면 됩니다.

## ⑤ 결정하고 다른 KG로 넘어가기

추천가를 따라 입찰가를 정했다면 끝! 다른 키워드그룹도 같은 방식으로 보면 됩니다. 빠른 검색에 다른 이름을 치고 ↑ ④ 단계 반복.

# 2. 대시보드 한눈에 보기

| 항목 | 주기 |

# 3. 페이지별 자세한 사용법

## 3-1. 입낙찰 히스토리 (메인 화면)

> 다음 회차 입찰가를 결정하는 핵심 페이지입니다. 매번 들어오게 될 거예요.

### 화면 4영역 구성

![입낙찰 히스토리 전체](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/bid-overview.png)

위에서 아래로:

1. **필터바** — 제품·대분류·소분류·키워드그룹·기간 + 빠른 검색
2. **요약 카드** — 키워드그룹 이름 + 최근 낙찰가
3. **좌측 (통계 + 회차 테이블)** — 가중평균 배수·공실률·추천 입찰가·시간 역순 회차 데이터
4. **우측 (추이 차트)** — 시간에 따른 가격 변화 라인 그래프

### 필터바 사용 규칙

필터는 **왼쪽에서 오른쪽으로** 종속됩니다. 상위를 바꾸면 하위는 초기화돼요.

| 필터 | 옵션 | 기본값 | 비고 |
|------|------|--------|------|
| 제품 | 서칭뷰 / 신제품검색 | 서칭뷰 | 바꾸면 대분류 이하 초기화 |
| 대분류 | (전체) + 카테고리 | (전체) | 선택해야 키워드그룹 활성화 |
| 소분류 | (전체) + 하위 카테고리 | (전체) | 대분류 선택 후 활성화 |
| 키워드그룹 | (선택) + KG 목록 | (선택) | 대분류 미선택 시 비활성화 |
| 기간 | 6 / 12 / 24 / 52 회차 | 12 | 최근 N회차만 표시 |

### 빠른 검색의 마법

![빠른 검색 자동완성](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/quick-search.png)

대분류/소분류 거치지 않고 KG 이름·대분류·소분류 어느 쪽으로도 검색 가능합니다.

- 텍스트 입력 → 최대 12개 추천
- ↑ ↓ 키로 이동
- `Enter` 로 선택
- `Esc` 로 닫기
- 선택 시 URL의 `product/cat1/cat2/kg` 가 한 번에 채워짐

### 추천 입찰가 카드 자세히 보기

![추천 입찰가 카드](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/recommend-card.png)

이 카드는 다음을 알려줍니다:

| 항목 | 내용 |
|------|------|
| 가중평균 낙찰/최저 배수 | 최근 4회차 가중평균 (5·3·2·1). 1.0x에 가까우면 경쟁 약함, 2.0x 이상이면 경쟁 강함. |
| 공실 발생률 | 슬롯이 안 채워진 회차 비율 (202622회차부터 측정). 높으면 진입 기회 많음. |
| 추천 입찰가 (VAT-) | 다음 회차 권장 가격 범위 (하한~상한) |
| 과거 적중률 | 같은 전략을 과거 회차에 적용했을 때 적중한 비율 |
| 예상 실 지불액 | (신제품검색만) 2순위 가격제 기준 실 지불 예상액 |

> 💡 도움말 아이콘(?) 위에 마우스를 올리면 적용된 전략 이름과 계산 방식이 떠요.

### 회차 테이블

![회차 테이블](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/round-table.png)

시간 역순(최신이 위)으로 회차별 입찰 내역이 표시됩니다.

| 컬럼 | 의미 |
|------|------|
| 회차 | 회차번호 (예: `202622` = 2026년 22주차) |
| 집행기간 | `MM.DD ~ MM.DD` |
| 기준조회수 | 해당 KG의 정규 검색량 |
| 최저 (VAT-) | 네이버 시스템의 최저 입찰가 |
| 낙찰 (VAT-) | 실제로 낙찰된 가격 |
| 배수 | 낙찰 ÷ 최저 |
| 공실 | 채워지지 않은 구좌 수 (없으면 `-`) |
| 집행 브랜드 | 광고를 노출한 브랜드 (여러 개면 `/` 구분) |
| 상태 | 정기입찰 (파랑) / 재입찰 (초록) / 기타 |

**한 행을 클릭**하면 회차 상세 모달이 열려요. ↓

### 회차 상세 모달

![회차 상세 모달](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/round-detail.png)

이 모달은 그 회차에 **누가, 어느 슬롯에, 어떤 광고로** 들어왔는지 보여줍니다.

각 슬롯 카드의 정보:

- 슬롯 번호 (1·2·3) — 검색 결과 페이지의 위치
- 표시명 — 광고에 노출된 브랜드명
- 광고 타이틀 + 부 타이틀 + 본문 (본문은 신제품검색만)
- 사업자명 — 실제 사업자등록명
- 신뢰도 — 0.00 ~ 1.00 (1에 가까울수록 정확히 매칭됨)

**닫기**: `X` 버튼 / `Esc` 키 / 모달 바깥 클릭

### 추이 차트

![추이 차트](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/trend-chart.png)

우측에 sticky로 따라오는 라인 차트입니다.

- **점선 (회색)** = 최저 입찰가
- **실선 (초록)** = 낙찰가
- **X축** = 회차번호
- **Y축** = 금액 (천 단위 콤마)

마우스를 점 위에 올리면 Tooltip 으로 회차·기간·가격·집행 브랜드를 한 번에 확인할 수 있어요.

### 엑셀로 다운받기

![엑셀 다운로드 버튼](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/download.png)

회차 테이블 우상단의 **다운로드 버튼** 을 클릭하면 현재 KG 회차 내역을 `.xlsx` 파일로 받을 수 있어요.

- 파일명: `{KG이름}_회차내역_{YYYY-MM-DD}.xlsx`
- 시트 1 (회차 요약): 화면 테이블 그대로 (9개 컬럼)
- 시트 2 (집행사 상세): 회차 × 슬롯 펼친 데이터 (10개 컬럼)

> 💡 보고서 만들 때나 피벗 분석할 때 유용해요.

### URL로 화면 공유

브라우저 주소창의 URL에 모든 필터 상태가 들어 있어서, **URL만 복사해서 슬랙/메일로 보내면** 받은 사람이 동일한 화면을 볼 수 있어요.

```
/?product=SEARCHING_VIEW&cat1=금융&cat2=금융상품&kg=12345&last=12
```

| 파라미터 | 값 | 의미 |
|---------|-----|------|
| `product` | `SEARCHING_VIEW` / `NEW_PRODUCT` | 제품 코드 |
| `cat1` | 카테고리명 | 대분류 |
| `cat2` | 카테고리명 | 소분류 |
| `kg` | 정수 | 키워드그룹 ID |
| `last` | 6 / 12 / 24 / 52 | 표시 회차 수 |

## 3-2. 브랜드 점유

> 카테고리 단위로 "어느 브랜드가 어느 회차에 얼마나 깔렸나" 를 한눈에 보는 히트맵입니다.

![브랜드 점유 히트맵](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/brand-heatmap.png)

### 필터바

| 필터 | 옵션 | 기본값 |
|------|------|--------|
| 제품 | 서칭뷰 / 신제품검색 | 서칭뷰 |
| 대분류 | (전체) + 카테고리 | (전체) |
| 기간 | 6 / 8 / 12 / 24 회차 | 8 |

### 히트맵 읽는 법

- **행** = 브랜드 (알파벳 정렬, 좌측 고정)
- **열** = 회차번호 (오름차순)
- **셀** = 그 브랜드가 그 회차에 노출된 **KG 개수**
- **색상 강도** = KG 개수 많을수록 진한 파랑. 0이면 점(`·`) 표시.

### 셀에 마우스를 올리면

우측에 툴팁이 떠요. 그 셀이 어떤 KG들로 채워졌는지 정확히 알 수 있습니다.

```
삼성화재 다이렉트 · 202622회차 · 4개 KG
─────────────────────
서칭뷰 > 금융 > 금융상품 > 자동차보험
서칭뷰 > 금융 > 금융상품 > 운전자보험
서칭뷰 > 금융 > 금융기관 > 보험사
서칭뷰 > 금융 > 금융서비스 > 보험가격비교
```

### 활용 시나리오

- **신규 진입 카테고리 파악** — 우리 브랜드가 0이었던 카테고리에 새로 진입한 시점 찾기
- **경쟁사 변화 감지** — 어떤 브랜드가 갑자기 점유율을 늘렸는지 시각적으로 확인
- **신규 광고주 출몰 추정** — 새 브랜드가 어느 회차부터 등장했는지 확인

### URL 공유

```
/brand?product=SEARCHING_VIEW&cat1=금융&last=8
```

## 3-3. 브랜드 추적

> "이 광고주, 어디어디 깔려 있지?" 를 알고 싶을 때 쓰는 페이지입니다.

![브랜드 추적](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/brand-tracker.png)

### 2단 레이아웃

- **좌측**: 브랜드 목록 (검색 가능)
- **우측**: 선택한 브랜드의 상세 정보

### 좌측 브랜드 목록

상단 검색칸에 브랜드명 또는 호스트(도메인)를 입력하면 실시간 필터링됩니다.

각 행에는 다음이 표시돼요:

- 브랜드명 (표시명)
- 호스트 (도메인, 여러 개면 `+N` 표시)
- 노출횟수 (총 노출 카운트)
- KG 개수 (서로 다른 키워드그룹 개수)

### 우측 브랜드 상세

브랜드 행을 클릭하면 우측 패널이 갱신됩니다. 다음 정보가 표시돼요:

**통계 카드 4개**

| 항목 | 의미 |
|------|------|
| 총 노출 횟수 | 누적 노출 카운트 |
| 누적 KG 수 | 서로 다른 키워드그룹 개수 |
| 회차 수 | 노출된 서로 다른 회차 개수 |
| 제품군 | 서칭뷰 / 신제품검색 |

**노출 타이틀 목록**

# 4. 자주 쓰는 시나리오

> 일하다가 "이럴 땐 어떻게 하지?" 싶을 때 보세요.

## 시나리오 A — 다음 주 입찰가 결정 ⭐ 가장 많이 함

이 흐름이 가장 빈번해요. 도식으로 정리하면:

![다음 입찰가 결정 흐름](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/flow-bid-decision.svg)

**단계별 설명**

1. **입낙찰 히스토리** 메뉴 진입
2. 빠른 검색에 KG 이름 입력 → 선택
3. **추천 입찰가 카드** 확인
4. **과거 적중률** 보고 결정:
   - 🟢 80% 이상 → 그대로 채택
   - 🟡🔴 80% 미만 → 회차 테이블·차트 추세 보고 보정
5. **공실 발생률** 체크 → 높으면 추천가 하한 근처, 낮으면 상한 근처

## 시나리오 B — 특정 회차의 경쟁 상황 파악

1. 입낙찰 히스토리에서 KG 선택
2. 회차 테이블에서 궁금한 회차 **행 클릭**
3. 모달에서 슬롯별 광고 카드 확인 (어느 브랜드가 어느 슬롯에 어떤 타이틀로 들어갔는지)

## 시나리오 C — 경쟁사 출몰 추적

1. **브랜드 점유** 메뉴 진입
2. 카테고리 선택, 기간 12회차 이상으로 늘리기
3. 히트맵에서 특정 시점부터 진하게 색이 채워진 브랜드 찾기
4. 셀 hover로 어느 KG에 들어갔는지 확인

## 시나리오 D — 특정 광고주 전체 활동 보기

1. **브랜드 추적** 메뉴 진입
2. 좌측 검색칸에 광고주명 또는 호스트 입력
3. 우측 패널에서 통계 카드 → 노출 타이틀 → 상세 테이블 순으로 확인

## 시나리오 E — 엑셀로 받아서 보고서 만들기

1. 입낙찰 히스토리 → KG 선택
2. 회차 테이블 우상단 **다운로드 버튼** 클릭
3. 받은 `.xlsx` 의 "회차 요약" 시트를 보고서에 붙여넣기
4. "집행사 상세" 시트로 슬롯별 분석 (피벗 테이블 권장)

## 시나리오 F — 동료에게 화면 공유

1. 원하는 필터/KG 상태를 만든다
2. **브라우저 주소창 URL** 복사
3. 슬랙·메일로 그대로 전달
4. 받은 사람이 클릭하면 동일한 화면이 뜸 (로그인 필요)

# 5. 용어 사전

> 처음 보는 용어가 많으시죠? 회차·KG·슬롯 관계를 그림으로 먼저 보면 이해가 빨라요.

![용어 관계도](https://nosp-dashboard-sgkim-dots-projects.vercel.app/guide/terms-relation.svg)

위 그림을 머릿속에 새기면, 다음 용어들이 자연스럽게 연결됩니다.

| 용어 | 풀어서 설명 |
|------|------------|
| **회차** | 네이버 검색광고의 입찰 사이클 단위. 보통 1주일 단위. `202622` 라고 쓰여 있으면 "2026년 22주차" 라는 뜻이에요. 매주 새 회차가 열려요. |
| **KG (키워드그룹)** | 비슷한 검색 의도를 가진 키워드 묶음. 예: "자동차보험" 이라는 KG 안에 "자동차 보험", "자동차보험 비교", "자동차보험료 계산" 같은 검색어들이 들어 있어요. 입찰은 KG 단위로 진행됩니다. |
| **최저가 (VAT-)** | 네이버 시스템이 정한 그 회차의 최저 입찰가. 부가세 별도. 입찰자는 이 가격 이상으로 써야 해요. |
| **낙찰가 (VAT-)** | 실제로 낙찰된 가격. 가장 높은 금액을 쓴 광고주가 슬롯을 차지해요. 부가세 별도. |
| **낙찰 배수** | `낙찰가 ÷ 최저가`. 1.0x = 최저가 그대로 낙찰. 2.0x = 두 배에 낙찰. 경쟁 강도를 한눈에 알 수 있어요. |
| **슬롯 (구좌)** | 검색 결과 페이지에서 광고가 노출되는 자리. 보통 1·2·3번 슬롯이 있어요. |
| **공실 / 공실 슬롯** | 그 회차에 채워지지 않은 슬롯. 입찰자가 없거나 가격이 안 맞은 경우입니다. 다음 회차에 "재입찰" 로 채워질 수 있어요. |
| **정기입찰** | 일반적인 다음 회차 입찰. |
| **재입찰** | 공실이 발생한 슬롯을 추가로 채우기 위한 입찰. |
| **집행 브랜드** | 그 회차에 실제로 광고를 노출한 브랜드. |
| **호스트 (Host)** | 광고주의 도메인. 예: `samsungfire.com` |
| **신뢰도 (Confidence)** | 브랜드명 자동 추출의 정확도 점수 (0.00 ~ 1.00). 1에 가까울수록 정확합니다. 0.8 이상이면 매우 신뢰 가능. |
| **출처 (Source)** | 브랜드 정보를 어디서 얻었는지: `dom` (광고 카드 자체) / `landing` (랜딩페이지) / `manual` (수기 매핑) |
| **추천 입찰가** | 최근 4회차 가중평균(`5·3·2·1`)을 기반으로 시스템이 제안하는 다음 회차 입찰가 범위. |
| **과거 적중률** | 같은 전략을 과거에 거꾸로 적용했을 때, 추천가 범위에 실제 낙찰가가 들어간 비율. |
| **예상 실 지불액** | (신제품검색 전용) 2순위 가격제 기준으로 실제 차감될 예상 금액. |
| **VAT-** | 부가세 별도 금액. 실제 청구 시 10% 추가됩니다. |
| **DPR / 캡쳐 시각** | 브랜드 정보를 스크래핑한 시점. |

# 6. 자주 묻는 질문 (FAQ)

### Q1. 키워드그룹을 골라도 데이터가 안 나옵니다.

- 그 KG가 신규로 만들어졌거나 아직 첫 회차가 시작 전이라 비어있을 수 있습니다.
- 회차는 있는데 브랜드만 "미수집" 이면 스크래퍼가 아직 처리 중이에요. 1~2일 후 다시 확인.
- URL의 `cat1` / `cat2` 값과 KG가 맞지 않으면(잘못된 조합) 데이터가 안 나옵니다. 필터바 좌측부터 다시 선택해 보세요.

### Q2. 최근 낙찰가와 추천 입찰가가 너무 다릅니다.

- 추천 입찰가는 단순 최근가가 아니라 **최근 4회차 가중평균** 으로 계산됩니다.
- 직전 회차에 일회성 이벤트(과도한 경쟁자 진입 등)가 있었다면 최근 낙찰가가 평균보다 높을 수 있어요.
- 차트의 직선 추세를 보고 추천가의 위치를 판단하세요.

### Q3. "미수집" 상태가 며칠째 안 바뀝니다.

- 광고가 실제로 노출되지 않았던 회차일 수 있어요 (집행사 없음 = 공실).
- 회차 행을 클릭해서 모달에서 "집행사 없음" 으로 표시되면 공실이 확정된 것입니다.
- 그래도 의심되면 sgkim@madup.com 으로 회차번호 + KG 이름을 알려 주세요.

### Q4. 브랜드 점유 히트맵이 비어있어요.

- 해당 카테고리에서 그 기간 동안 광고가 한 번도 노출되지 않은 경우입니다.
- 기간을 더 길게(예: 24회차) 늘려보거나 카테고리를 (전체)로 풀어보세요.

### Q5. 신뢰도가 0.5 정도면 믿어도 되나요?

- 신뢰도는 자동 추출 결과의 확신도입니다. 0.8 이상은 매우 정확, 0.5 ~ 0.8 은 참고용, 0.5 미만은 의심해 봐야 합니다.
- 동일 브랜드의 여러 노출 기록을 교차 확인하면 정확도를 높일 수 있어요.
- 사업자명·호스트명이 함께 매칭되면 신뢰도가 낮아도 실제로는 정확할 가능성이 높습니다.

### Q6. 엑셀 다운로드가 작동 안 합니다.

- 다운로드 버튼은 회차 데이터가 1개 이상 있을 때만 활성화됩니다. 키워드그룹을 먼저 선택했는지 확인하세요.
- 브라우저 팝업 차단이 켜져 있으면 다운로드가 막힙니다. 주소창의 차단 아이콘에서 허용해 주세요.
- 파일명에 특수문자(`/`, `\`, `:`, `*` 등)가 있으면 자동으로 `_` 로 치환됩니다.

### Q7. 화면을 동료에게 공유하고 싶어요.

- 원하는 필터 상태를 만든 후 브라우저 주소창 URL을 그대로 복사하세요.
- 받은 사람이 그 URL을 열면 동일한 화면이 뜹니다 (로그인 필요).
- 동료가 아직 가입하지 않았다면 회사 이메일로 가입하도록 안내해 주세요.

### Q8. 모바일에서도 보이나요?

- 보이긴 하지만 가로 스크롤이 많아서 권장하지 않습니다.
- 데스크탑(1280px 이상) 환경에서 최적화되어 있어요. 차트와 사이드 패널이 나란히 보이려면 1536px 이상을 권장합니다.

### Q9. 데이터 갱신 주기는?

- 매주 새 회차가 추가됩니다 (네이버 입찰 사이클에 맞춰).
- 브랜드 정보(집행사)는 회차 시작 후 스크래퍼가 순차적으로 채웁니다. 통상 첫 1~2일 안에 대부분 채워져요.
- 추천 입찰가는 새 회차가 들어오는 즉시 다시 계산됩니다.

### Q10. 권한·계정 문제는 어디로?

- 가입·로그인 안 됨, 다른 사람 계정 추가 등 운영 문의: **sgkim@madup.com**
- 가급적 회사 이메일로 가입하세요. 외부 이메일은 추가 승인이 필요할 수 있어요.
